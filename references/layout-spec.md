# Xiaohongshu Studio Layout Spec

这份规范描述当前 `xhs-studio.html` 的真实布局和分页约束。它不使用独立 JSON 画布模型。

## Canvas

- 默认尺寸：`1080 × 1440`
- 固定比例：`3:4`
- 默认左右边距：`72px`
- 默认上边距：`72px`
- 默认下边距：`72px`
- 默认正文字号：`36px`
- 默认正文行高：`58px`（`36px` 字号 + `22px` 行间留白）
- 默认段间距：`40px`
- 同一序列内部项目间距：`20px`；序列整体与相邻结构的段间距仍为 `40px`
- 默认封面分割线：画布高度的 `50%`
- 一级标题保留原编号、字号与引线样式；两位编号使用统一的固定栏宽、等宽数字特性并在栏内居中，编号与标题保持固定间隔。其引线上沿为 `174px`。横线纸/方格纸每 `58px` 一条/格，第 3 条/格与该引线对齐；蓝图格每 `43.5px` 一格，第 4 格与该引线对齐。
- 二级标题使用 `36px` 正文规格的强调色文字与 `2px` 下划线；正文结构块统一采用仅保留下间距的单向节奏：普通二级标题上间距为 `0px`、下间距为 `40px`，页首作为第一个块时上间距为 `44px`，下划线与 `174px` 引线对齐。一级标题、二级标题、引用、卡片、序列、图片等相接时只由前一块提供下间距，避免相邻样式的上下间距叠加。

可使用其他接近 3:4 的尺寸，例如：

- `900 × 1200`
- `1242 × 1656`
- `1242 × 1660`

同一篇笔记的每页必须使用相同尺寸。

## Content Flow

页面不是相互独立的文本框。所有正文块先组成连续流，再按浏览器真实高度分页。普通正文按**字符**切开跨页，下一页从断点处一字接续；只有源稿切行（空行、Enter、Shift+Enter）或结构块边界才允许断流。

Enter 切分和 reflow 必须是可验证事务：切分前后正文逐字等价；重排前后正文、图片、手动空行与分页符的顺序签名一致。校验失败时回滚，不得保存可能丢字的状态。

段间距只属于两个相邻块之间。分页测量时，页面最后一个块的底部段间距不占用可用高度；只要图片、卡片、引用或其它完整块的可见本体能放入安全边距，就应保留在当前页。

用户按 Enter 创建的手动空行是连续流中的真实节点，但当它恰好成为页首或页尾时必须折叠为 `0px`：既不丢失空行编辑语义，也不能让下一页第一行先出现一行空白。正文末尾空白区的点击与拖放只显示临时 gap/drop cursor，不写入空段；松手后按新的区块顺序重新分页并回填页面。图片落到页尾空白时允许把图片框高度收至可用空间，避免区块顺序已经变化但图片仍回到下一页；该适配只能修改图片框高度，不能生成空段或删除后续正文。

### 可拆分

- 普通正文段落
- 富文本正文
- 超长卡片正文（保留首段角标，后续页不重复角标）

### 默认保持完整

- 编号子标题
- 引用块
- 普通卡片
- macOS 风格代码块
- 有序 / 无序序列
- 单图和图片组

如果单个图片块高于整页，编辑器可自动缩小未被用户手动设置高度的图片框。

## Cover

转换器接受 `--cover-image <file>`。图片会以 data URL 嵌入首张封面的上半页，使用 `cover` 裁剪；因此保存后的独立 HTML 不依赖原图的本地路径。

### 显示图片

- `0–50%`：封面图片。
- `50–100%`：主标题、副标题和装饰线。

### 关闭图片

- `0–50%`：主标题和副标题。
- `50–100%`：连续正文流的开头。

封面正文尾区必须保持源顺序。第一个放不下的块出现后，不允许跳过它去放后面的短块。

## Text

- 字体固定使用书刊宋体栈：转换前通过 `scripts/ensure-font.js` 确保安装第一优先级 `Noto Serif SC`，随后依次回退 `Source Han Serif SC`、`Noto Serif CJK SC`、`Songti SC`、`STSong`、`SimSun`。不得增加操作系统私有 `STSongti-*` 映射。
- 正文默认纯黑，普通正文、序列、卡片与表格正文全部统一使用 `720` 加粗字重；源 Markdown 的 `**加粗**` 与 Studio 手动加粗保留语义标签，但视觉字重同样为 `720`，不再出现较淡的 `700` 正文。Markdown 加粗不得自动叠加下划线；下划线只保留原稿明确样式或 Studio 手动操作。一级标题、封面标题和大编号保留标题专用字重。
- 正文左对齐，末行同样左对齐。
- 有序序号在独立编号栏内居中，与正文固定间隔 `9px`。
- 子标题的编号与标题文字是同一结构块，分页时一起移动。
- 封面主标题最多三行，按可用区域自动减小字号。
- 封面副标题最多两行，并按中英文权重限制长度。

## Images

- 图片块默认与正文内容宽度一致并居中。
- 图片框为直角，白色底。
- `contain` 用于完整显示，`cover` 用于裁剪填满。
- 图片内容通过 `object-position` 和 transform 保存裁剪中心与缩放。
- 图片框宽高与图片内容缩放是两个独立参数。
- 相邻竖图最多自动组成三图组；用户手动拆分后不应再次强制合并。

## Editor Caret

图片、卡片、引用、列表和子标题前后需要可插入文字的光标锚点。锚点必须：

- 在视觉上仅悬停时提示。
- 不占分页高度。
- 点击后可直接输入。
- 自动 reflow 重建页面后恢复到原字符位置并继续保持编辑区焦点。
- 保存和导出时从 DOM 副本中移除。

## Export

- 导出使用页面 DOM 的独立副本，不直接截图带选中框的编辑画布。
- 导出前去除阴影、选中框、缩放控制点和光标锚点。
- 主题 CSS 变量、图片裁剪参数和行内样式必须复制到导出副本。
- 下划线在导出前需要稳定化，避免被 `html2canvas` 渲染成整块背景或单独细线。
- 页面先按 `2x` 画布渲染，再使用高质量图像平滑缩回目标尺寸，改善宋体笔画边缘，同时保持最终文件仍为标准尺寸。
- 默认导出 `1080 × 1440` PNG，文件按页码排序。
