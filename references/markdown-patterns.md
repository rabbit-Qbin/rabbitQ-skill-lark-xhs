# Markdown Patterns for rabbitQ-skill-lark-xhs

这份约定用于准备飞书导出的 Markdown。转换器会独立解析 Markdown，不需要先经过公众号 HTML。

## 推荐结构

```markdown
---
title: "AI 视频一眼假怎么破？"
subtitle: "这次不是讲概念，是一次真实踩坑复盘。"
author: "小兔Q彬"
tags: "AI视频,Seedance,Prompt"
---

# AI 视频一眼假怎么破？

这次的核心结论是：不是模型不行，而是 **提示词太完美**。

## 01 问题长什么样

- 皮肤过于平滑
- 动作太稳定
- 表情没有真实人的小瑕疵

> 引用块适合放需要停下来读的观点。

**金句：小红书不是论文，用户先看懂，才有后面的转化。**

## 02 我怎么改

把「完美真人」改成「有轻微瑕疵的真实拍摄」。

![修改前后对比](图片和附件/image.png)
```
## 标题

- H1 用作封面标题。
- H2-H6 用作正文编号子标题。
- 已写 `01`、`02` 的标题会保留编号；没有编号时自动连续编号。
- 编号和标题属于同一个块，不能通过空行拆散。

## 正文与行内样式

- 普通段落会成为可按行跨页的正文。
- `**加粗**` 保留为粗体。
- 整段加粗且长度足够时，可自动推断为卡片。
- 只有明确以“金句 / 结论 / 注意 / 划重点”等标签开头的部分加粗段落，才按卡片处理。

## 引用

```markdown
> 引用块适合放观点、说明和补充背景。
```

飞书原生引用导出后通常也是 `>`。引用结构优先于关键词，即使内容出现“金句”也不会变成卡片。

## 列表

```markdown
- 无序项目 A
- 无序项目 B

1. 有序项目 A
2. 有序项目 B
```

列表结构优先于内容关键词。列表项中出现“卡片”“重点”等词，仍保持列表。

## 表格

推荐使用标准 GFM pipe table：

```markdown
| 模式 | 适合 | 页数 |
| --- | --- | ---: |
| 有封面图 | 视觉强、产品感 | 6 |
| 无封面图 | 干货长文、教程 | 8 |
```

- 转换器通过“表头行 + `---` 分隔行”识别表格，不会把普通竖线文本误判为表格。
- 支持对齐冒号、首尾省略 `|`，以及单元格内用 `\|` 表示竖线。
- 飞书导出为 HTML `<table>` 时也会保留行列。
- 建议控制在 2–5 列；列太多应先精简，不要依赖横向滚动。

## 图片

```markdown
![图片说明](图片和附件/image%201.png)
```

- 推荐把 Markdown 和附件目录一起传入转换器。
- 支持中文、空格和 URL 编码路径。
- 连续竖图可能自动并排。

## 视频

视频不会进入小红书图文：

```markdown
演示视频.mov
```

转换器会跳过视频并生成提示。需要视频内容时，先截帧为图片，或在小红书发布页单独上传。

## 不要手工分页

不要插入 `<!-- xhs-page -->` 或通过大量空行控制页码。Studio 会按实际字体、行距、图片高度和 3:4 版心自动分页。需要人为调整时，优先修改内容、图片框高度或块顺序。
